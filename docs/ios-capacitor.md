## Architecture

### Build-Time Toggle: `VITE_CAPACITOR`

The env var `VITE_CAPACITOR=true` controls two things at build time:

| | Web (`VITE_CAPACITOR` unset) | Native (`VITE_CAPACITOR=true`) |
|---|---|---|
| **Vite base path** | `/chataiagent/` | `/` |
| **PWA plugin** | Enabled (service worker, manifest) | Disabled |
| **Router basename** | `/chataiagent` | `/` |
| **PWAInstall component** | Rendered | Skipped |
| **API URL** | From `.env.local` | Hardcoded production URL |

### Runtime Platform Detection

```ts
// src/lib/capacitor.ts
import { Capacitor } from '@capacitor/core';
export const isNativePlatform = Capacitor.isNativePlatform();
```

Used in:
- `src/App.tsx` â€” dynamic `basename` + conditional `<PWAInstall />`
- `src/config/firebase.ts` â€” `initializeAuth` with `browserLocalPersistence` for WKWebView
- `src/components/chat/streaming-response.tsx` â€” YouTube thumbnails instead of iframes

## iOS-Specific Fixes

### 1. Firebase Auth in WKWebView

WKWebView has IndexedDB issues that break Firebase's default persistence. Fix:

```ts
// src/config/firebase.ts
export const auth = isNativePlatform
  ? initializeAuth(app, { persistence: browserLocalPersistence })
  : getAuth(app);
```

### 2. YouTube Embeds

Iframes don't load in WKWebView. Native builds show clickable thumbnails that open YouTube app:

```ts
// src/components/chat/streaming-response.tsx
if (native) {
  // Show thumbnail with play button overlay â†’ opens YouTube app/Safari
} else {
  // Web: use iframe embed
}
```

### 3. Radix ScrollArea Horizontal Overflow

Radix renders `<div style="display:table; min-width:100%">` inside the viewport, which allows content to push wider than the screen:

```css
/* src/index.css */
[data-radix-scroll-area-viewport] > div {
  display: block !important;
  min-width: 0 !important;
  max-width: 100% !important;
}
```

### 4. iOS Keyboard Detection

The `useIOSKeyboard` hook uses the Visual Viewport API to detect keyboard height:

```
src/hooks/use-ios-keyboard.ts
```

- 150px threshold to distinguish keyboard from URL bar changes
- `requestAnimationFrame` for smooth updates
- Handles orientation changes with 300ms delay

### 5. Safe Area Handling

```css
/* src/App.css */
#root {
  padding-top: env(safe-area-inset-top, 0px);
}
.ios-input-container {
  padding-bottom: env(safe-area-inset-bottom);
}
```

Combined with `contentInset: 'automatic'` in Capacitor config.

### 6. Scroll Containment

Multiple layers prevent iOS rubber-band bounce from leaking:

- `html, body { overscroll-behavior: none; overflow: hidden; }`
- ScrollArea viewport: `overscrollBehavior: 'contain'`
- Chat content: `.chat-content { overscroll-behavior: contain; }`

## Troubleshooting

### App shows blank white screen
- Run `npm run build:ios && npm run cap:sync` to refresh web assets
- Check Xcode console for JavaScript errors
- Verify `ios/App/App/public/index.html` exists and has correct asset paths (`/assets/...` not `/chataiagent/assets/...`)

### Auth fails with "internal-error"
- Firebase IndexedDB issue in WKWebView â€” ensure `firebase.ts` uses `browserLocalPersistence` for native

### Chat content clips horizontally
- Verify Radix ScrollArea CSS override in `index.css`
- Check `overflow-x: hidden` on scroll viewport

### Keyboard covers input bar
- Verify `Keyboard.resize: 'native'` in `capacitor.config.ts`
- Check `ios-input-container` class has `padding-bottom: env(safe-area-inset-bottom)`

### YouTube videos don't play
- Expected: iframes don't work in WKWebView
- Should show thumbnail with play button that opens YouTube app

### Build fails in Xcode
- `npm run cap:sync` after any web code changes
- Check Xcode â†’ Signing & Capabilities â†’ Team is set
- Check `ios/App/CapApp-SPM/Package.swift` resolves all plugin packages
